## Asset result

The release filter is evaluated from the extracted
`DT_PalMonsterParameter` and icon table. It excludes boss, raid boss, tower
boss, quest, summon, police, Oilrig, Gym, Paldex-unreleased, icon-missing,
and configuration-incomplete rows. It selects 288 forms without consulting an
auxiliary implementation.

All 41,616 unordered parent pairs are generated. The Cat Mage / Fox Mage pair
has two gender orientations, producing 41,617 logical rows. The compact site
table decodes back to the independently regenerated asset rows with zero
missing rows, extra rows, duplicates, unreleased IDs, or result differences.

## Native static findings

Byte-verified regions of the exact server executable establish that normal
child selection excludes `IsBoss`, `IgnoreCombi`, and unique-combination
children. `IgnoreCombi` is not checked while reading either parent's rank.
Normal selection uses the ceiling average rank, minimum absolute distance,
higher `CombiDuplicatePriority`, and then source row enumeration order. No
rarity or variant flag tie-break is present in the inspected routine. The
native function can add a clamped breeding-item `CombiRankBonus`, but all four
entries extracted from `DA_BreedingItemEffectData` have value zero, so every
available item uses the base ceiling-average target in this build.

Two unreleased quest rows pass the native child filter, but lose their ties to
earlier released base rows and change no public-parent result. The complete
top-level orchestration region checks special combinations and, on a miss,
immediately enters normal selection. It contains no equal-parent shortcut.
Applying that verified control flow to all 288 self pairs yields 286 identity
results plus `KingWhale + KingWhale -> DomeArmorDragon` and
`PlantSlime_Flower + PlantSlime_Flower -> PlantSlime`. The first result also
provides the one output-level counterexample for omitting `IgnoreCombi`.
Neither exceptional parent form is produced by another released-parent pair,
so reverse lookup correctly has zero candidates for those two forms.

## Auxiliary comparisons

PalCalc, PalworldSaveTools, and Paldeck are pinned by commit and file SHA-256.
They are comparison-only inputs. PalCalc covers all 41,617 logical rows but its
equal-parent shortcut disagrees on the two native self-pair results (41,615
matching rows, two missing and two extra rows). PalworldSaveTools and Paldeck
each omit the newly released form and its 288 pairs; PalworldSaveTools also has
the covered `KingWhale` difference, while Paldeck has that difference plus one
collapsed gender-dependent result. None of these projects can add, remove, or
change a site row.

## Unverified boundary

The result table covers the base released-form ID. Boss/Alpha rows and later
individual-state processing are outside its 288-form scope. The top-level
native breeding function has not been directly invoked for every pair, and the
41,616 pairs have not been exhaustively observed as in-game hatches. The site
therefore describes the result as a fixed-build asset table with native static
evidence, not as an official or exhaustive in-game verification.
