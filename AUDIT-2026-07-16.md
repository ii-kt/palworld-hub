## Asset result

The release filter is evaluated from the extracted
`DT_PalMonsterParameter` and `DT_PalCharacterIconDataTable` fields. It first
excludes boss flags, non-positive Paldex entries, missing icons, and incomplete
parameters. Rows sharing the same `(ZukanIndex, ZukanIndexSuffix, Rarity)`
asset-form key are resolved to the single row with its own exact character-icon
ID; RowName prefixes and suffixes are never classified. This removes 11
duplicate internal parameter rows across nine form keys and retains the one
asset-distinct tribe-icon fallback form, selecting 288 forms without consulting
an auxiliary implementation.

All 41,616 unordered parent pairs are generated. The Cat Mage / Fox Mage pair
has two gender orientations, producing 41,617 logical rows. The compact site
table decodes back to the independently regenerated asset rows with zero
missing rows, extra rows, duplicates, unreleased IDs, or result differences.

## Native static findings

Byte-verified regions of the exact server executable establish that normal
child selection excludes `IsBoss`, `IgnoreCombi`, and unique-combination
children. `IgnoreCombi` is not checked while reading either parent's rank.
Normal selection uses the ceiling average rank, minimum absolute distance,
higher `CombiDuplicatePriority`, and then source row enumeration order. No
rarity or variant flag tie-break is present in the inspected routine. The
native function can add a clamped breeding-item `CombiRankBonus`, but all four
entries extracted from `DA_BreedingItemEffectData` have value zero, so every
available item uses the base ceiling-average target in this build.

Two unreleased quest rows pass the native child filter, but lose their ties to
earlier released base rows and change no public-parent result. The complete
top-level orchestration region checks special combinations and, on a miss,
immediately enters normal selection. It contains no equal-parent shortcut.
Applying that verified control flow to all 288 self pairs yields 286 identity
results plus `KingWhale + KingWhale -> DomeArmorDragon` and
`PlantSlime_Flower + PlantSlime_Flower -> PlantSlime`. The first result also
provides the one output-level counterexample for omitting `IgnoreCombi`.
Neither exceptional parent form is produced by another released-parent pair,
so reverse lookup correctly has zero candidates for those two forms.

## Native runtime result

GitHub Actions run `30018308091` at main commit
`9c79baa3f1f3ddda60f20a79399a1d4c91fb5f14` downloaded the public dedicated
server, then rejected it unless the executable and PAK matched the pinned byte
counts and SHA-256 values above. With Internet egress blocked, the audit
harness injected rows built from the exact extracted
`DT_PalMonsterParameter` and `DT_PalCombiUnique` evidence and called the fixed
native function at `0x71168c0`.
The harness instruments the raw and unique-table `FindRow` paths, row-key
generation, manager `FindRow`, and manager helper; only the game's selection,
special-recipe, gender, and tie-break code remains unmodified. It is therefore
a native-function comparison against injected table rows, not a live-table
integration test.

Every 41,616 unordered pair was invoked in both parent orders and both
meaningful gender orientations: 166,464 native calls in total. All 41,617
logical rows matched. Runtime row metadata, unique rows, logical results,
individual calls, parent order, hidden gender orientation, same-species
control flow, special combinations, normal selection, and boss/Alpha species
mapping each had zero differences. The JSON evidence file SHA-256 is
`265bf315873f9d4f1e58ac8fec9544b912e7e6cea304cdc3b34cb1437be63bb1`;
its canonical self-digest is
`08d7850d2bb566a77cd8734c93b7ed8f31563c287850e41450de2328c89a36a6`.
No executable, PAK, Steam data, or compiled probe was uploaded or committed.

## Auxiliary comparisons

PalCalc, PalworldSaveTools, and Paldeck are pinned by commit and file SHA-256.
They are comparison-only inputs. PalCalc covers all 41,617 logical rows but its
equal-parent shortcut disagrees on the two native self-pair results (41,615
matching rows, two missing and two extra rows). PalworldSaveTools and Paldeck
each omit the newly released form and its 288 pairs; PalworldSaveTools also has
the covered `KingWhale` difference, while Paldeck has that difference plus one
collapsed gender-dependent result. None of these projects can add, remove, or
change a site row.

## Unverified boundary

The result table covers the base released-form ID. The exhaustive native call
used exact extracted tables injected by the audit harness; it did not read the
live PAK DataTable through an unmodified manager. Boss/Alpha species mapping
was checked, but later individual-state processing is outside the 288-form
table. The 41,616 pairs have not been exhaustively observed as in-game hatches.
The site therefore does not describe the result as an official or exhaustive
in-game hatch verification.
